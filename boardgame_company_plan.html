Step 0 — Game Selection

Target Games: Not Popular Yet in Spain

Código Secreto
Social Deduction · 2–8 players

The physical version is known in Spain but has no dominant web implementation in Spanish. Pure word game, low legal complexity, huge social appeal.

★ Priority 1 Low complexity
Sushi Go!
Card Drafting · 2–5 players

Barely known in Spain. Extremely teachable, fast rounds (15 min), works perfectly async online. BGA has it but no Spanish-first experience.

★ Priority 1 Underserved
Hanabi
Cooperative · 2–5 players

Almost unknown in Spain. Unique mechanic (you can't see your own cards). Perfect for web — the limitation is a feature online. Strong virality potential.

★ Priority 1 Hidden gem
Just One
Party / Cooperative · 3–7 players

Spiel des Jahres winner. Cooperative word game where duplicate answers cancel out. No strong Spanish web version. High accessibility for non-gamers.

Priority 2 Mass market
Sea Salt & Paper
Card Game · 2–4 players

Rising star in Europe, very visual, fast (20 min). Perfect mobile/web translation. Not yet digitized in Spanish. Recent enough to be trending.

Priority 2 Trending
Skull
Bluffing · 2–6 players

The purest bluffing game. 6 pieces per player, zero luck claims. Social dynamics make it addictive. Almost zero digital presence in Spain.

Niche but sticky
Kingdomino
Tile Placement · 2–4 players

Great family gateway game. Spiel des Jahres 2017. Visual and easy to grasp. Its tile placement mechanic translates beautifully to web interfaces.

Priority 2
Love Letter
Deduction / Elimination · 2–6 players

Only 16 cards. Extremely fast (5 min/round). Perfect for mobile async play. Classic bluffing with a tiny rules footprint. Very implementable.

Priority 3 Quick win
Coup
Social Deduction · 2–6 players

Hidden identity, bluffing, and elimination. Hugely popular in English-speaking markets. Almost untouched in the Spanish web market. Very fast to implement.

Some BGA presence Gap in Spanish
The Pipeline

Phase-by-Phase Process (Per Game)

01
Game Research & Viability
Market fit · Legal analysis · Complexity scoring
Research Agent
+
  • BGG rating + review mining for Spain/Europe sentiment
  • Check if game has existing digital version (BGA, Steam, App Store)
  • Evaluate digital translation difficulty (tokens, maps, hidden info)
  • Check license/IP status — public domain vs publisher deal needed
  • Estimate player count, session length, async compatibility
  • Spanish community research (Reddit, Discord, forums)
  • Score game on: viral potential, complexity, licensing risk
  • Competitive gap analysis vs BGA and existing apps
🤖 Agent: ResearchBot — scrapes BGG, App Store reviews, Google Trends (Spain), and outputs a structured viability report.
02
Licensing & Legal Clearance
Publisher contact · IP strategy · Terms of use
Legal/Ops Agent
+
  • Contact publisher for digital licensing (e.g. Repos Production for Just One)
  • Draft non-commercial fan version if publisher permits free play
  • Revenue-share agreement drafting for commercial games
  • GDPR-compliant user data policy
  • Terms of Service and privacy policy per game/region
  • Trademark check for game name and UI elements
🤖 Agent: LegalDraftBot — templates licensing emails, tracks publisher response status, flags games cleared for development.
03
Game Design Document (GDD)
Rules engine · State machine · Edge cases
Design Agent
+
  • Full rules formalization as state machine (JSON schema)
  • Player state, game state, action definitions
  • Edge case catalogue (ties, disconnections, illegal moves)
  • Async turn model design (push notifications, turn timers)
  • AI opponent design if applicable (difficulty levels)
  • Spectator mode rules
  • Tutorial flow design (interactive first-time UX)
🤖 Agent: GDDBot — takes physical rulebook as input, outputs structured GDD with state machine, action list, and edge case log.
04
UI/UX Design System
Unified brand · Per-game theming · Accessibility
Design Agent
+
  • Global design system: typography, tokens, spacing, shadows
  • Shared component library: buttons, modals, player panels, chat
  • Per-game theme layer (Sushi Go warm/playful vs Hanabi dark/elegant)
  • Responsive layout for mobile, tablet, desktop
  • Accessibility audit (WCAG AA): contrast, keyboard nav, screen readers
  • Animation system: game events, turn transitions, win animations
  • Dark mode support per game
  • Figma/design token pipeline to code
🤖 Agent: DesignBot — generates Tailwind design tokens, component stubs, and theme variants from a brand brief.
05
Software Architecture
Backend · Real-time · Database · Infra
Dev Agent
+
  • Monorepo structure: shared game engine + per-game frontend
  • Game engine as pure TypeScript logic (no side effects, fully testable)
  • WebSocket server for real-time game state sync
  • REST API for auth, lobbies, history, leaderboards
  • PostgreSQL: users, games, sessions, events log
  • Redis: ephemeral game state, presence, matchmaking queue
  • Auth: Supabase or Auth.js (email + Google + guest mode)
  • Deployment: Vercel (frontend) + Railway/Fly.io (backend)
  • CDN for static assets, CI/CD via GitHub Actions
🤖 Agent: ArchBot — scaffolds new game repos from GDD, wires game engine to API, generates boilerplate tests.
06
Implementation
Game engine · Frontend · Multiplayer · AI
Dev Agent
+
  • Pure game engine: all rules, scoring, win conditions (TDD)
  • React/Next.js frontend per game with shared component library
  • WebSocket integration: optimistic UI updates + server reconciliation
  • Lobby system: public/private rooms, invite links, async mode
  • Tutorial mode: step-by-step interactive walkthrough
  • Basic AI opponent (rule-based first, MCTS optionally later)
  • Spanish + English i18n from day one
  • PWA manifest: installable on mobile, offline lobby browsing
🤖 Agent: DevBot — writes game engine from GDD spec, flags ambiguous rules, runs tests automatically.
07
QA & Playtesting
Automated tests · Human playtest · Balance
QA Agent
+
  • Unit tests: 100% coverage on game engine logic
  • Property-based tests: fuzzing against random game states
  • E2E tests: Playwright scripts simulating full multiplayer games
  • AI vs AI simulations: run 10,000 games, check score distributions
  • Human playtest sessions: 5+ groups, 3+ sessions each
  • Session recording analysis: where do players get confused?
  • Performance testing: 100+ concurrent games on staging
  • Mobile UX audit: touch targets, scroll behavior, orientation
🤖 Agent: QABot — generates test suites from GDD, runs AI simulations, reports broken rules + performance bottlenecks.
08
Monetization Integration
Freemium · Subscriptions · Cosmetics
Growth Agent
+
  • Stripe integration for subscription and one-time payments
  • Free tier gate design (e.g. 3 async games/day, limited cosmetics)
  • Premium tier: unlimited async games, all cosmetics, no ads
  • Cosmetics shop: card backs, player avatars, table themes
  • Referral system: invite friend → unlock cosmetic
  • A/B test paywall placement and premium messaging
  • Publisher revenue share tracking dashboard
🤖 Agent: MonetizationBot — tracks conversion funnel, suggests gate adjustments based on drop-off data.
09
Launch & Community
SEO · Social · Influencers · Spain-first
Growth Agent
+
  • Spanish board game community outreach (Reddit r/juegosdemesa, Discord servers)
  • Spanish gaming influencers and streamers (Twitch/YouTube)
  • Product Hunt / BoardGameGeek listing
  • SEO: Spanish-language landing pages per game
  • Email waitlist → launch announcement sequence
  • Press kit and coverage in Spanish gaming media
  • Launch event: online tournament with prize cosmetics
🤖 Agent: GrowthBot — monitors community mentions, tracks keyword rankings, manages email campaigns.
10
Operations & Iteration
Analytics · Feedback · Roadmap updates
Ops Agent
+
  • PostHog or Plausible analytics: session funnels, feature flags
  • Error monitoring: Sentry with game-context tagging
  • Weekly metrics report: DAU, conversion rate, churn, game popularity
  • User feedback form in-game → tagged Notion/Linear backlog
  • Quarterly game drop: new game added every 3 months
  • Balance patches based on simulation data
  • Community voting on next game to implement
🤖 Agent: OpsBot — generates weekly health reports, triages feedback into priority buckets, flags anomalies in game data.

Agent Architecture

AI Agent Pipeline

🔎
ResearchBot
Game Intelligence Agent

Monitors BGG, App Store reviews, Google Trends Spain, and tabletop forums to produce structured game viability reports.

  • Weekly scan of trending games in Europe
  • Scores new candidates on 6 criteria
  • Alerts team when a game crosses viability threshold
  • Tracks competitor (BGA, Steam) new game launches
📐
GDDBot
Rules Formalization Agent

Takes physical rulebook PDFs or text as input and produces a structured Game Design Document with state machine, action schema, and edge case catalogue.

  • Parses rules into JSON state machine
  • Flags ambiguous or missing rules
  • Generates test case seeds from edge cases
  • Outputs GDD Markdown for developer handoff
⚙️
ArchBot
Game Scaffolding Agent

Scaffolds a new game repository from a GDD. Creates the game engine skeleton, API routes, WebSocket events, and frontend component stubs.

  • Generates typed game engine from GDD schema
  • Wires to shared multiplayer infrastructure
  • Bootstraps per-game design token theme
  • Creates CI/CD pipeline config for new game
🧪
QABot
Testing & Simulation Agent

Generates unit and E2E tests from GDD, runs AI-vs-AI game simulations, and flags logic bugs or score imbalances before human playtesting.

  • Auto-generates test suites from GDD actions
  • Runs 10k AI simulation games per build
  • Detects score distribution anomalies
  • Flags broken game states in CI pipeline
📊
OpsBot
Operations & Analytics Agent

Generates weekly health reports, monitors error rates, triages user feedback into actionable backlog items, and detects anomalies in production data.

  • Weekly KPI digest (DAU, conversion, churn)
  • Sentry error triage and prioritization
  • Feedback categorization → Linear issues
  • Revenue report with publisher share breakdown
📣
GrowthBot
Marketing & Community Agent

Monitors Spanish-language board game communities, tracks keyword rankings, manages email sequences, and identifies influencer opportunities.

  • Monitors r/juegosdemesa and Discord servers
  • Tracks SEO rankings for target keywords
  • Drafts community announcements and patch notes
  • Manages referral and waitlist email flows
